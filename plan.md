# 记作业助手开发计划

## 1. 项目目标

开发一个面向大学生的记作业助手，用于快速记录老师写在黑板上的作业信息。用户可以打开应用后直接拍照记录黑板内容，并在保存时选择学科、截止日期和截止时间。应用首页以看板形式展示所有作业，未完成作业优先展示，并按照截止时间先后排序；已完成作业展示在下方，方便学生清晰管理学习任务。

## 2. 目标用户

- 大学生
- 需要同时管理多门课程作业的学生
- 习惯通过拍照记录课堂信息的学生
- 容易忘记作业截止时间、需要统一看板提醒的学生

## 3. 核心使用场景

1. 老师在黑板上布置作业。
2. 学生打开“记作业助手”。
3. 点击“拍照记录作业”。
4. 调用摄像头拍摄黑板内容。
5. 拍照完成后进入作业保存表单。
6. 选择已有学科，或快捷新建一个学科。
7. 选择作业截止日期和截止时间。
8. 可选填写备注。
9. 点击保存后，作业进入首页看板。
10. 看板中未完成作业优先展示，并按截止时间从近到远排列。
11. 作业完成后，用户右划作业卡片即可标记为已完成。
12. 已完成作业自动移动到列表下方。

## 4. 功能范围

### 4.1 作业看板

- 展示所有作业记录。
- 未完成作业始终优先展示。
- 未完成作业按照截止时间从近到远排序。
- 已完成作业展示在未完成作业下方。
- 已完成作业可视觉弱化，降低干扰。
- 支持空状态提示。
- 支持临近截止时间提示。

作业卡片展示内容：

- 学科名称
- 截止时间
- 作业照片缩略图
- 备注内容
- 完成状态
- 临近截止提示

### 4.2 拍照记录作业

- 首页提供“拍照记录作业”按钮。
- 调用 `uni.chooseImage` 打开摄像头。
- 拍照后展示照片预览。
- 用户可以确认保存，也可以取消重新拍摄。
- 照片路径随作业记录保存到本地。

### 4.3 学科选择与管理

- 应用本地维护一个学科表。
- 初次使用时提供默认学科。
- 保存作业时可以直接从已有学科表中选择。
- 学科选择区域提供“快捷新建学科”入口。
- 用户输入学科名称后：
  - 将新学科加入学科表。
  - 自动选中新建学科。
  - 将学科列表保存到本地存储。
- 后续新增作业时可以继续选择已创建学科。
- 学科名称为空时不允许创建。
- 已存在同名学科时不重复创建。

默认学科建议：

- 高等数学
- 大学英语
- 程序设计
- 数据结构
- 线性代数

### 4.4 截止时间选择

- 保存作业时必须选择截止日期。
- 保存作业时必须选择截止时间。
- 截止日期和截止时间组合为完整截止时间。
- 截止时间用于看板排序和临近截止判断。
- 展示格式建议为：`YYYY-MM-DD HH:mm`。

### 4.5 标记完成

- 作业卡片支持右划操作。
- 当右划距离超过阈值时，将作业标记为已完成。
- 标记完成后立即更新本地数据。
- 已完成作业自动移动到未完成作业下方。
- 已完成作业记录完成时间。

### 4.6 本地数据持久化

- 使用 `uni.setStorageSync` 和 `uni.getStorageSync` 保存数据。
- 应用关闭后再次打开，作业和学科仍然保留。
- 作业数据和学科数据分开存储。

建议存储 Key：

- `homework_list`：作业列表
- `subject_list`：学科列表

## 5. 数据结构设计

### 5.1 作业数据结构

```js
{
  id: string,
  subject: string,
  deadline: string,
  deadlineDate: string,
  deadlineTime: string,
  imagePath: string,
  note: string,
  completed: boolean,
  createdAt: string,
  completedAt: string | null
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 作业唯一 ID |
| `subject` | `string` | 所属学科名称 |
| `deadline` | `string` | 完整截止时间，用于排序 |
| `deadlineDate` | `string` | 截止日期 |
| `deadlineTime` | `string` | 截止时间 |
| `imagePath` | `string` | 拍摄的作业照片路径 |
| `note` | `string` | 备注内容 |
| `completed` | `boolean` | 是否已完成 |
| `createdAt` | `string` | 创建时间 |
| `completedAt` | `string | null` | 完成时间 |

### 5.2 学科数据结构

```js
{
  id: string,
  name: string,
  createdAt: string
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 学科唯一 ID |
| `name` | `string` | 学科名称 |
| `createdAt` | `string` | 创建时间 |

## 6. 排序规则

作业看板排序规则：

1. 未完成作业排在已完成作业前面。
2. 未完成作业按截止时间从近到远排序。
3. 已完成作业按完成时间从近到远排序。
4. 若已完成作业缺少完成时间，则按创建时间从近到远排序。

示例逻辑：

```js
const sortedList = homeworkList.sort((a, b) => {
  if (a.completed !== b.completed) {
    return a.completed ? 1 : -1
  }

  if (!a.completed && !b.completed) {
    return new Date(a.deadline) - new Date(b.deadline)
  }

  return new Date(b.completedAt || b.createdAt) - new Date(a.completedAt || a.createdAt)
})
```

## 7. 页面设计

### 7.1 首页结构

首页文件：`pages/index/index.vue`

页面包含：

- 顶部欢迎区
- 作业统计区
  - 未完成数量
  - 已完成数量
  - 临近截止数量
- “拍照记录作业”主按钮
- 新增作业表单
- 作业看板列表
- 空状态提示

### 7.2 新增作业表单

表单字段：

- 作业照片预览
- 学科选择
- 快捷新建学科输入框
- 截止日期选择器
- 截止时间选择器
- 备注输入框
- 保存按钮
- 取消按钮

表单校验：

- 未拍照时不允许保存。
- 未选择学科时不允许保存。
- 未选择截止日期时不允许保存。
- 未选择截止时间时不允许保存。

### 7.3 作业卡片样式

作业卡片应具备：

- 清晰的学科标签。
- 醒目的截止时间。
- 图片缩略图。
- 简短备注展示。
- 未完成状态突出显示。
- 已完成状态降低透明度或显示完成标识。

### 7.4 临近截止提示

建议规则：

- 截止时间距离当前时间小于 24 小时，显示“即将截止”。
- 截止时间已过且未完成，显示“已逾期”。
- 普通未完成作业不显示强提示。

## 8. 交互设计

### 8.1 拍照流程

1. 用户点击“拍照记录作业”。
2. 应用打开摄像头。
3. 用户完成拍照。
4. 页面显示照片预览和保存表单。
5. 用户补充学科、截止时间和备注。
6. 用户点击保存。
7. 作业进入看板。

### 8.2 快捷新建学科流程

1. 用户在学科选择区域点击“新建学科”。
2. 输入学科名称。
3. 点击确认。
4. 系统判断是否为空或重复。
5. 校验通过后加入学科表。
6. 自动选中新建学科。

### 8.3 右划完成流程

1. 用户在作业卡片上向右滑动。
2. 系统判断滑动距离是否超过阈值。
3. 超过阈值则标记为已完成。
4. 更新本地存储。
5. 看板重新排序。
